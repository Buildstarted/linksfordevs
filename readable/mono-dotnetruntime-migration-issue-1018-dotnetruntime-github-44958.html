These are the high level sets of tasks / priorities leading up to and after parts of mono are migrated into dotnet/runtime.

Before Migration

  • Determine steps needed to enable dotnet/runtime testing for mono
  • Enable PR Sync Bot and test

Migration

  • Execute mono consolidation script (requires mono merge freeze)
  • Merge PR into dotnet/runtime: #1912
  • Enable PR Sync Bot

Post Migration

Priority I (by Feb 1st tentatively)

  • Build MonoVM at the same time as CoreCLR
  • Enable LLVM support + binary packages: #2140
  • Adjust, if necessary, helper targets like run-sample run-benchmark: #2393
  • Migrate ios and android build/test targets

Priority II

  • Meta-package for LLVM dependencies
  • Migrate wasm build/test targets
  • Get ios and android test targets running on devices / simulators
  • Run mono/tests tests and mono/mini tests on MonoVM on CI
  • Run MonoVM's soft debugger tests on CI (this will need some coordination with https://github.com/mono/debugger-libs)

Is there a plan to port Mono to use the .NET Core GC?

Xamarin and Blazor are the primary workloads for Mono. CoreCLR GC (as is) is likely a worse option for these workloads than the current Mono GC. We may experiment with it, but the Mono GC is here to stay until we have a data that confirm the .NET Core GC would be a better choice for the primary Mono workloads.
