GitHub draft pull requests should trigger the pipeline

by Add your solution
Reading time: 1-2 minutes
Posted here: 06 Aug 2020

https://developercommunity.visualstudio.com/content/problem/1140412/github-draft-pull-requests-should-trigger-the-pipe.html

Sam Harwell [MSFT] reported 37 minutes ago

Starting earlier this week, draft pull requests on GitHub stopped triggering our build pipelines. This broke our main use cases for draft pull requests:

  • Draft pull requests are used to validate the build on multiple operating systems
  • Draft pull requests are used to validate tests on multiple operating systems

After the above are complete and the build results show that a change passes, a pull request is marked ready for review. If the triggers for builds are not automatic, users must remember to manually trigger the build for every single change. For us, this is equivalent to not having automatic build triggers at all for pull requests.
