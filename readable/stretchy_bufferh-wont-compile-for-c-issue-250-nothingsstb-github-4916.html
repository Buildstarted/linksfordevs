std::is_pod is pretty useless IMHO.

struct vec3 {
    float x,y,z;
    vec3(float x_, y_, z_) : x(x_), y(y_), z(z_) {}
    vec3() : x(0.0f), y(0.0f), z(0.0f) {};
};

is not POD according to std::is_pod, because it has constructors - even though they don't do any magic (like having side-effects outside the struct or even allocating memory).

Putting that in a stretchy buffer is totally safe.

So yeah, a short note should probably be added, but people should figure out themselves what's safe and what is not.
Also I think, if you're writing actual C++ code you should use std::vector or something - making this work with C++ is mainly useful for C-code that you want to be able to compile as C++ as well (like in header-only libs).
