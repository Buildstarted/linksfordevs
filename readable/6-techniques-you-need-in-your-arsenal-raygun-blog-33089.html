Tip 4: Freeze multithreaded applications

Next on our list of .NET debugging tips, we‚Äôre going to talk about multi-threaded applications.

Debugging multi-threaded applications can be a real pain, especially if multiple threads are executing the same code. Wouldn‚Äôt it be great if it were possible to tell Visual Studio that you want to debug just this one thread and not switch to other threads that are currently executing the same code?

Well, turns out, there is a way to do this.

Open the Threads window via Debug -> Windows -> Threads. This window shows all threads of the current application. Double-clicking a thread switches to it. Right-clicking a thread shows different options, one of which is Freeze:

As the name suggests, this freezes the execution of that particular thread. To use this feature in our scenario, you can set a breakpoint at the beginning of the code that you want to debug.

When the breakpoint is hit, use the Threads window to freeze all threads for the current location except one. You can now debug this one thread and single-step through the code without thread switches interrupting your concentration. If you want to continue running the program after you‚Äôve debugged the code in the thread, you should unfreeze‚Äîor Thaw‚Äîthe other threads so they can perform their work as well.
