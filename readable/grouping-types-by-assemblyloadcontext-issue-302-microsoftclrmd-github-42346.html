Grouping types by AssemblyLoadContext · Issue #302 · microsoft/clrmd

by raffaeler
Reading time: 1 minute
Posted here: 18 Nov 2019

https://github.com/microsoft/clrmd/issues/302

In the .NET Framework, types can be loaded in different AppDomains.
Now with .NET Core, they can instead be loaded in different AssemblyLoadContexts. Since any outstanding reference to an AssemblyLoadContext can prevent it to be unloaded, it becomes fundamental to diagnose which are the types being loaded in a specific context.

Currently I am diagnosing this kind of issue by analyzing the roots of System.Reflection.LoadAllocator, but this is not always enough, since I have to search the first root living in a different load context.

Is there any way to identify the Load Context of the objects?
