Understanding Functions in JavaScript

A step-by-step guide to getting familiar with JavaScript functions.

In JavaScript, functions are being used with increasing frequency. If you’re familiar with JavaScript development then you know how important these things are when working on a development project. Therefore, It’s a good idea to understand the functions clearly before you use them.

Functions

In each programming language, we can see a different way of using these functions. Before we move to an example of JavaScript functions, let’s try to understand what’s a function and why we need functions in JavaScript.

A function is a set of statements which we use to perform a particular task and the importance of using a function is, we can write it once and use it as many times we want.

For instance, you can write a simple function that gets two numbers as parameters and return the sum of those two numbers. So, you can use this function anywhere in your program when you need to add two numbers and get the total in return. Remember that you have to call the function whenever you need it.

In JavaScript, we can write a function in 3 different ways.

i. Regular Function -Function declaration

In a function declaration, you can use the function keyword and write it as follows.

function welcome(name){console.log('Hello ', name);}

In the above method, we start our function using the function keyword, then write the name of the function. If you take any parameters for your function, as the above example place them inside the parentheses. If it doesn’t get any parameters then leave the parentheses empty.

Then open a bracket and write your code inside it. When you need to invoke this function, type welcome('John') wherever you need.

ii. Function Expression

Basically, in function expression what we do is store a function inside a variable. In JavaScript, we use var, let and const to declare variables. So, when it comes to function expression, we can write the above function as below.

const welcome = function(name){console.log('Hello ',name);};

When you call this function, use the variable name welcome and place parentheses at the end. ex: welcome('John').

By looking at these two methods of function declarations, you could think it only differs from the syntaxes. But when it comes to Hoisting in JavaScript, there’s a considerable difference between these two methods.

If you use the first way of function declaration, you can either declare the function before you call it or after you call the function. Let’s try to understand this using the below example.

ex 01:welcome('John');/*
  some code here
*/function welcome(name){
console.log('Hello ',name);
}welcome('John');

Here, this function call works either when you place it before the function declaration or after the declaration.

But when it comes to function expression, you can’t call the function before you declare it. Always you need to declare the function first and then you can call it.

ex 02:const welcome = function(name){
console.log('Hello ',name);
};welcome('John');

iii. Arrow Function

This is known as the shorter form of the function declaration. Let’s try to understand this with the below code sample.

// function expression
const welcome = function(name){
console.log('Hello ',name);
};// arrow function
const welcome = name => {
console.log('Hello ',name);
};

Here, the function keyword is not needed and you can directly place parameters you use for the function. If you’re using only a single parameter for the function as above (ex: name) you don’t need to surround the parameter with parenthesis.

But, if you’re using two or more parameters for the arrow function, it’s essential to surround them using a parenthesis.

ex 03:// arrow function which takes two parameters const msg = (name, time) => {
console.log(`Hey ${time}, ${name}`);
};msg('John','good morning'); 
//note: order of the argument should be matched with the parameters.

Also, if you’re not taking any parameters to your function, keep the parenthesis empty as below.

ex 04:// arrow function with no parametersconst msg_2 = () => 'hey there!';console.log('Message is: ',msg_2());

Let’s try to be more familiar with the arrow functions using the below example.

ex 05: // suppose we need to find the area of a triangle by giving height and base valuesconst calcArea = (height, base) => {
return (height * base)/2;
};const area = calcArea(10, 12);
console.log('Area is: ',area);// since you're returning a value in a single line, you can simplify this arrow function further as belowconst calcArea = (height, base) => (height * base)/2;

Conclusion

In most occasions, developers tend to use function expression and arrow function to declare functions in JavaScript.

If you’re still new to JavaScript, go with the regular function declaration and once you are getting used it gradually, you will find that function expression and arrow functions are easier to use than the regular method.
