Exploring ASP.NET Core 3.0

Reading time: 1-2 minutes
Posted here: 12 Nov 2019

https://andrewlock.net/series/exploring-asp-net-core-3/

In this series I explore some of the changes and new features coming to ASP.NET core 3.0. I'm not going to be looking at the big headline features, like server-side Blazor or the gRPC client factory, as there's already lots of posts on those.

Instead, I'm going to look at some of the more minor features and more subtle changes. Some of them these changes will likely pass you by unnoticed, others could bite you if you're not aware of them, and others provide some nice new features.

Posts in this series (new posts will be listed here as they're written):

  1. Part 1 - Exploring the new project file, Program.cs, and the generic host
  2. Part 2 - Comparing Startup.cs between the ASP.NET Core 3.0 templates
  3. Part 3 - New in ASP.NET Core 3: Service provider validation
  4. Part 4 - Running async tasks on app startup in ASP.NET Core 3.0
  5. Part 5 - Introducing IHostLifetime and untangling the Generic Host startup interactions
  6. Part 6 - New in ASP.NET Core 3.0: structured logging for startup messages
  7. Part 7 - New in .NET Core 3.0: local tools
