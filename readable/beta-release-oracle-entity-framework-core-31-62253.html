Beta Release: Oracle Entity Framework Core 3.1

by https://medium.com/@alex.keh
Reading time: 3 minutes
Posted here: 29 May 2020

https://medium.com/@alex.keh/beta-release-oracle-entity-framework-core-3-1-eb0b4d3c93b1

I’m happy to announce the release of Oracle Entity Framework Core (EF Core) 3.19.0 beta on NuGet Gallery. This beta supports the new changes in Entity Framework Core 3.1. Since it’s a beta, be sure to check off the “Include Prerelease” box when searching for the assembly on NuGet Gallery.

No Oracle EF Core app code changes are necessary to upgrade from EF Core 2.x to the Oracle EF Core 3.1 beta and EF Core 3.1. You should be able to drop the assembly into an existing Oracle EF Core 2.x app, upgrade to EF Core 3.1 and the beta, then run.

My team’s dev work focused on supporting the EF Core 3.x changes Microsoft required of provider writers. Those changes are within the provider itself, such as how LINQ translates to Oracle SQL and maintaining compatibility with a wide range of Oracle Database versions. The beta supports Oracle Database server 11.2.0.4 and higher, on-premises or in the cloud.

That’s not to say there will be no issues for existing apps when upgrading to EF Core 3.x. Microsoft has documented the breaking changes that occur when moving from EF Core 2.x to 3.x.

The Oracle EF Core 3.1 documentation is part of the NuGet package. Unzip the package to retrieve the PDF file or find it installed in your NuGet global packages directory. For example:

C:\Users\<username>\.nuget\packages\Oracle.EntityFrameworkCore\3.19.0-beta1

The Oracle EF Core 3.1 documentation generally covers the changes since the Oracle EF Core 2.x release. You can find the Oracle EF Core 2.x doc here.

To submit feedback and bugs, post to the ODP.NET discussion forum. Before reporting a bug, please check the “Limitations and Known Issues” doc section first.

For the beta, we focused on feature completeness, compatibility, and quality. There are no additional features planned for production at this time. The production release timing depends on beta feedback. If developers are not seeing many issues, the beta period could be fairly short. I encourage you to submit feedback at your earliest opportunity. Simple and complete test cases when reporting bugs are appreciated.

Just a couple more notes. This beta doesn’t support .NET 5. Also, EF Core 3.0 is not supported as Microsoft has desupported the version.

Try out the beta and let us know what you think!
