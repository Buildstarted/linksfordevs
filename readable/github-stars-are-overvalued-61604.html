GitHub stars are overvalued

by https://towardsdatascience.com/@calebkaiser
Reading time: 5-6 minutes
Posted here: 12 May 2020

https://towardsdatascience.com/github-stars-are-overvalued-15ba780b36?gi=dd747808c0ea

There’s more to an open source project than its star count

Intuitively, we all know that GitHub stars are not make-or-break for an open source project. Clicking the “Star” button simply says, “I find this interesting.” Not a bad signal to collect, but not the last word on whether or not a project is successful.

Despite this, stars are enticing. They’re simple, countable, and signal positive feedback. As a user, a high star count gives you a reason to check out one project over another. As a maintainer, every bump in stars gives you a little boost of dopamine—and I’m speaking from experience here.

I work on Cortex, open source machine learning infrastructure, and despite knowing better, I still get irrationally excited by surges in stars.

Apparently, I’m not alone. There is an entire ecosystem built around stars. Open source marketing has become its own niche.

Open source teams dedicate announcements to star milestones. A whole suite of tools has emerged for tracking and analyzing star growth (see the chart above).

Somewhere along the line, stars went from “a public way to support a project” to “the currency of open source software,” and it’s worth reevaluating how much a star really counts for.

Stars do certainly have value—more on that later—but the emphasis on them today is out of proportion. In general, stars are not as valuable as we tend to treat them, for a few reasons.

1. Stars aren’t equivalent to users

You can star a project without using it. You can use a project without starring it. This might feel obvious, but it’s important to underscore that stars and users don’t share a necessarily causal relationship (though, of course, one can lead to the other).

At Cortex, for instance, a surprising number of users haven’t starred the repo.

Our north star for evaluating whether or not Cortex is healthy is the number of people using it to deploy models in production. We measure that by interacting with actual users. If that number is going up, we know we’re doing something right. If it’s going down, we have some issues to address, even if the star count is increasing.

2. Stars are highly dependent on context

Stars are not a consistent unit of value, and comparing star counts between projects isn’t completely straightforward.

The size and activity of a project’s community, its number of potential users, and the marketing efforts of the maintainers have a massive effect on a project’s star count.

If you are a frontend Javascript framework and you have Facebook’s marketing budget, for example, you’re going to have more stars than a tool for DevOps engineers. In other words, the fact that React has 148,000 stars compared to Elasticsearch’s 49,000 doesn’t say much about whether or not Elasticsearch is a success—they’re apples and oranges.

3. Stars are still fundamentally a social media engagement

GitHub obviously isn’t Twitter, but stars are still a social media engagement, and all the usual caveats apply.

For starters, some GitHub users are more active on the social side than others. Some users star 2,000 projects, others barely star any.

Similarly, many GitHub users use stars to signal general support for a project, even if they never intend to use it. When we first launched Cortex, for example, we received a lot of stars from the Go community, even from people who weren’t interested in machine learning. There’s nothing wrong with that—and we’re very grateful for the support—it just reinforces that stars are disconnected from usage.

Despite the above caveats, getting a star is still a good thing (in fact, if you want to give Cortex a star, please don’t let me stop you).

Stars are a signal that people are generally interested in a project, and in the same vein, can provide a useful benchmark between similar projects. It might not make any sense to compare React and Elasticsearch, but the fact that React has over twice the stars of Angular is probably meaningful.

Stars also provide some feedback as to who finds a project interesting. With Cortex, we were somewhat surprised to see how popular the project was among developers, not just data scientists. This influenced quite a few decisions in how we’ve developed the project.

For all of these reasons, stars are valuable—they just aren’t a complete metric for evaluating an open source project.
