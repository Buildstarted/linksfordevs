This appears to be the same problem as https://github.com/dotnet/corefx/issues/34202#issuecomment-450589549

@EatonZ the issue boils down to the signature not having a fully complete PKCS#1.5 signature that includes the OID. CNG is pickier about the signature missing that OID, while CAPI allows for it. CNG does support it, but you need to pass options that .NET Core or Framework do not expose at the moment.
