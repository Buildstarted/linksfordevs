.NET Transactional File Manager Migrated to GitHub – Chinh Do

by By Chinh Do
Reading time: 2 minutes
Posted here: 25 Mar 2020

https://www.chinhdo.com/20200321/net-transactional-file-manager-migrated-to-github/

I have migrated the .NET Transactional File Manager project to GitHub from CodePlex. I originally wrote about it in this blog post.

The library allows you do include file I/O operations in your transactions using System.Transactions. For example you can wrap a SQL Server insert and creation of a new file on disk in the same transaction, and commit or rollback the two operations together as a unit.

Example code:


TxFileManager fileMgr = new TxFileManager();
using (TransactionScope scope1 = new TransactionScope())
{
    
    fileMgr.CopyFile(srcFileName, destFileName);

    
    dbMgr.ExecuteNonQuery(insertSql);

    scope1.Complete();
}

This library is available as NuGet package. I just published version 1.4.0-beta1 there with the following fixes:

  • Add support for custom temp paths to address issues with file/dir operations accross filesystems
  • Fix for resource leak in TxFileManager._txEnlistment
  • Target .NET Standard 2.0
