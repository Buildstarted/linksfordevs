Core 3.1 release shows as Microsoft .NET Core Host - 3.1.0 Preview 3 (x86) · Issue #3962 · dotnet/core

by zulander1
Reading time: 1-2 minutes
Posted here: 06 Jan 2020

https://github.com/dotnet/core/issues/3962

Thanks for filing this. It was also reported in the 3.1.0 announcement post (#3950) and there are some details about what's going on in a related issue (dotnet/runtime#492), but it might be good for this problem to have its own dedicated issue.

An important detail about this bug is the hosting bundle didn't actually install the 3.1.0 preview 3 build from last month. What's being installed is a more recent build that happens to have a 3.1.0 preview 3 label.

The .NET Core Runtime version in the rightmost column shows you have the newest version available right now:

  • 3.1.0.28312 is the newest build, close to 3.1.0. (This is in the screenshot, thanks for including it!)
  • 3.1.0.28303 was the version of the 3.1.0 preview 3 build released a month ago.

You can also check to see what you have installed with dotnet --info, which should show Microsoft.NETCore.App 3.1.0, rather than the old 3.1.0-preview3.19553.2.

We're still investigating this issue.
