Stop The Overuse Of Copy-Pasting And Unecessary Libraries

Reading time: 3-4 minutes
Posted here: 15 Jul 2020

https://edvinleander.com/2020/07/15/stop-the-overuse-copy-pasting-and-unecessary-libraries/

DISCLAMER! This is my opinions which some may find controversial. The point of this article is to start a debate and perhaps give you some revelations. Comments and further debates in the comment-section is appreciated. Enjoy the article!

Copy | Paste

Copy-Pasting Saves Time

Yes, that is true. And for some code snippets there is no problem with just copy-pasting it. Short snippets which you understand can be beneficial to your learning experience. I myself is guilty of copying code from Stack Overflow and other blogs. But those code snippets were just boilerplate which I understood. So, yes, copy-pasting saves time.

Copy-Pasting Carelessly is a Bad Idea

I know this. You are stressed and have tried to solve one problem for hours. What should I do, I cannot figure it out! Or maybe this one: I am a newbie and just want to get code so I can continue with my program. Both issues can be solved with copy-pasting without giving it some thought.

Onwards to my point! When you copy and paste you cement those things in your program. A function builds on another function and at the end you may have an unstable fundament. If you understand and can freely morph the copied code, you are in the clear. But if you just pasted it to continue the program and 50 functions later come to the realization that you must change that function. Can you? What is the answer? Copy-paste more? No, no, no … Now you have trouble! If you are not comfortable with the code, do not use it. Using code without understanding it are a ticking timebomb.

Libraries

Libraries Saves Time

If you do everything yourself (you are a badass) you are going to spend a lot of time reinventing the wheel. I myself are one of those people that wants to do everything myself. I want control! But if it is one thing I hate, it is boilerplate code. Libraries can help with that, making the boring past easy. So libraries can save time.

Overuse of Libraries is a Bad Idea

Hold your horses! Now you better have a great argument (the angry mob lights the torches). Using a library which is well supported and rigorously tested. Nothing wrong here… But can you really know every part of a library or framework, do you know the code? This problem is what I want to address in this text. If you do not know what is going on inside this black box of a library, how can you optimize it to run better and faster? Using a library can help you speed up the development. But at what cost? Overuse of libraries when you can spend time on creating it yourself is a problem.

Going Further

I know that this will either aggravate you or support your ideas. I wrote this text because I want to increase awareness of proper coding. So next time you use a library or copy-paste something, do what you think is right, now you know some advantages and drawbacks.
