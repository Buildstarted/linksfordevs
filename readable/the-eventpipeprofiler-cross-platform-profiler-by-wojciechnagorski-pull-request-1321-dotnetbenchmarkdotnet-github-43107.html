The EventPipeProfiler cross-platform profiler by WojciechNagorski · Pull Request #1321 · dotnet/BenchmarkDotNet

by WojciechNagorski

https://github.com/dotnet/BenchmarkDotNet/pull/1321

Your suggestion look good but it doesn't have the same function as my constructor.

You can call my constructor with parameters:

EventPipeProfiler(
    EventPipeProfile.CpuSampling, 
    new[] {
        new EventPipeProvider("System.Buffers.ArrayPoolEventSource", EventLevel.Informational, Int64.MaxValue),
    });

It give you all providers from EventPipeProfile.CpuSampling and add the System.Buffers.ArrayPoolEventSource provider.
In this way, you can also override the provider from the profile. e.g.:
the EventPipeProfile.CpuSampling profile adds:

new[]
{
    new EventPipeProvider("Microsoft-DotNETCore-SampleProfiler", EventLevel.Informational),
    new EventPipeProvider("Microsoft-Windows-DotNETRuntime", EventLevel.Informational, (long) ClrTraceEventParser.Keywords.Default)
}

and you can override for example EventLevel or change keywords, using the following code

EventPipeProfiler(
    EventPipeProfile.CpuSampling, 
    new[] {
        new EventPipeProvider("Microsoft-DotNETCore-SampleProfiler", EventLevel.Verbous),
    });

Currently it works in the same way like dotnet trace. You can fine more information here https://docs.microsoft.com/en-us/dotnet/core/diagnostics/dotnet-trace#options-1

But if you want I will change it. Please let me know.
