Strange float performance issues near zero · Issue #1308 · dotnet/BenchmarkDotNet

by AndreyAkinshin
Reading time: 2-3 minutes
Posted here: 13 Nov 2019

https://github.com/dotnet/BenchmarkDotNet/issues/1308

Wasn't sure where else to ask this and I figured a lot of people here have experience with benchmarks showing odd behaviors.

In short, I have a benchmark that does nothing more than multiple a vector float by a const value of 0.016. After several iterations of the benchmark, it becomes almost 10x slower. I must be something to do with the float approaching zero but I have no idea. I couldn't find anything on google that would explain a linear float operation not being a constant operation.

Here is a couple line project that reproduces this issue (the README shows a benchmark log). https://github.com/xposure/FloatPerfIssue

Could anyone provide any insight as to what is happening in .NET or at the CPU level?
