It also strikes me that this encoding is similar to the one @LukaJCB worked on for error elimination for similar reasons (attempting to describe a bi polymorphic F[_, _] in terms of two mono polymorphic F[_] and G[_]). The trick is relating the bi encoding to the mono encoding through the laws.

What allows you to do this (in part) is a way to get from the "eliminated" effect to the "introduced" effect, so you can state something like accept(fa.provide(r)).provide(r) === fa.provide(r) (i.e. idempotent elimination).
