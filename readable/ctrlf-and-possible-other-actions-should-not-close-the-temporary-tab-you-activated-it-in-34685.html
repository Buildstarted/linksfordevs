Ctrl+F (and possible other actions) should not close the temporary tab you activated it in - Developer Community

Reading time: 2 minutes
Posted here: 29 Aug 2019

https://developercommunity.visualstudio.com/idea/717168/ctrlf-and-possible-other-actions-should-not-close.html

Nick Craver reported Aug 29, 2019 at 12:35 PM

Reproduction steps:

  1. Ctrl+T (Go to All)
  2. Type class name
  3. Click on result (this opens some sort of “temporary”/“preview” tab - purple in dark theme)
  4. Click in file even (optional, but illustrating the point of it being active/focused)
  5. Ctrl+F (to find in the file)
  6. File (temporary tab!) disappears.
  7. User is sad and misses their beloved file they wanted to search

I’m trying to search in the file I have open and my cursor is in. But the behavior is to close the file. Instead, I suggest the tab be made a “real” tab when you try to take an action inside the file. The current behavior of closing it is non-intuitive and I can’t see how it’s ever what a user wants when performing a per-file action. What file are they trying to target if not the open and focused one?

Can we please change the behavior here for a more intuitive and useful experience? I currently waste a lot of time re-doing the Ctrl+T, finding the file again, making sure it’s not a preview tab, and then re-doing my desired action.
