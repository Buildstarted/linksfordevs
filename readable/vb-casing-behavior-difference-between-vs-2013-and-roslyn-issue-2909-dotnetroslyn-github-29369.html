VB Casing behavior difference between VS 2013 and Roslyn · Issue #2909 · dotnet/roslyn

by gafter
Reading time: 1 minute
Posted here: 11 Jul 2019

[A customer reports in DevDiv 1172900]

I have a solution that I can build in VS 2013 without any problem. However, when I try and build the same solution in VS 2015 it will not compile.
I'm not 100% sure if this is a bug with the compiler or not. The project that is having trouble building in VS 2015 is a VB project. The code that is having an error is:

Case MySqlDbType.Datetime

The compile error is BC31429. The problem is that the MySql.Data dll that I am referencing contains an enum with the following two members:

MySqlDbType.DateTime 
MySqlDbType.Datetime

VS2013 seems to handle the case differences just fine even though VB is a case insensitive language. The VS2015 compiler, however, does not seem to handle the case differences the same way and gives me an ambiguity error.
