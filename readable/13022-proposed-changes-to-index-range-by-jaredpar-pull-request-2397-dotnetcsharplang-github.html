@lostmsu if they are structs, I kind of like this approach. Or at least, it would be nice to have this baked in.

As for your suggested "under the hood" conversion, that could be just an implicit conversion operator. No compiler magic required since it could be inlined using normal means.

Problem is that standard collection interfaces won't support these. Not an issue for array, span, string, etc., but there's not a good way for retrofit any new collection operators back into the old interfaces.

I sometimes have a similar operation implemented using something like your range proposal struct plus additional generic structs that also accepts a list, collection, array, etc. Eg. ArrayRange<T>, etc., which implement implicit conversion operations and support enumeration, and implement IReadOnlyCollection, IReadOnlyList, etc.
