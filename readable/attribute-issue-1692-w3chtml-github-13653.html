Proposal: <button once="true" /> attribute · Issue #1692 · w3c/html

by siusin
Reading time: 2 minutes

https://github.com/w3c/html/issues/1692

A common case I've seen many hundreds of times across the years is: clicking a <button> should disable it. This is usually due to an HTTP request (or some other operation) that's kicked off. Clicking twice is usually an error.

I propose something to more easily handle this case:

<button once="true">Foo</button>

This would be equivalent to:

<button onclick="this.disabled = true;">Foo</button>

As a bonus, we could consider a CSS addition along the lines of:

button:loading { // ...or button:disabled:loading perhaps?
  // spinny thing, but no seizures please!
}

(maybe that naming is terrible, but the idea is what I want to convey)

I'd love to have something like this. We could save so much JavaScript written and cases where it's not written continually caught and addressed. The user would only need to re-enable the button when the operation completes and it's valid to do so...the same thing they need to do today, no net change on the tail end.

In the non-JavaScript case...things just work. With no JS in play, we're in a full form submission/reload scenario, so this just lets you not even use JavaScript to protect the simple case. To me, this may be the strongest argument for such an attribute.

When building apps, I'd even go so far as to validate it (in our tooling). Having an attribute makes that easy to reason about and check for. A sea of JavaScript and possibly-attached handlers isn't, though to be fair inline we could check for.

Thoughts?
