Configure 3rd Party Libraries for Testing | bUnit

Reading time: 2 minutes
Posted here: 08 Jun 2020

https://bunit.egilhansen.com/docs/providing-input/configure-3rd-party-libs

Testing a component that is built using a 3rd party component library can require additional setup steps in each test, to ensure the 3rd party components work correctly in a test context.

If for example a 3rd party component requires services to be registered in production code, it is likely that this is needed in a test context as well. See the Injecting Services into Components Under Test page for more info on this.

Similarly, if a root component is needed to provide cascading values to the 3rd party components, that is likely to be needed as well. See the Passing Parameters to Components page for more on this.
