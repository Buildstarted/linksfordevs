Not only an option, this should be the default. It's actively harmful for projects to continue to build. When downstream projects try to build after a dependency has failed a lot of time is wasted.

The developer's time is wasted by the doomed build itself spending time and resources, but also by the onslaught of errors that are accumulated. Irrelevant errors. What mattered here? It could be a missing semi-colon, but in generally it's one to a few errors. But that's not what the user gets, they get dozens to hundreds of errors from a missing DLL, some new class downstream code needs that missing, or just entire missing namespaces. Whatever it is...it's all irrelevant and just adds insult to injury. Downstream projects should never keep building after we know they're going to fail. It's a waste of everything to do so.

Please, please make stopping the default behavior.
