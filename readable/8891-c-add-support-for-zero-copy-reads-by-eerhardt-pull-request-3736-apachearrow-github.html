[C#] Add support for zero-copy reads by eerhardt · Pull Request #3736 · apache/arrow · GitHub

@eerhardt, not directly related to your changes, but I've been trying them out, and one thing that's related to minimizing allocations that I bumped up against is that ArrowStreamWriter always disposes the underlying stream. That's very undesirable if the arrow data being written is just part of a larger chunk of data being written out, such as if there's a header and footer that need to wrap it, as it forces you to make a temporary copy in order to not dispose the actual stream you want to write subsequent additional data to. ArrowStreamWriter should have a ctor that allows for leaving the underlying stream open upon dispose, as exists for built-in .NET types like StreamWriter, Deflatestream, etc.
