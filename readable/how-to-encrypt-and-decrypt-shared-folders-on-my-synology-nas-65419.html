How to encrypt and decrypt shared folders on my Synology NAS

by Synology Inc.
Reading time: 12-15 minutes
Posted here: 09 Aug 2020

https://www.synology.com/en-global/knowledgebase/DSM/tutorial/File_Sharing/How_to_encrypt_and_decrypt_shared_folders_on_my_Synology_NAS

Data encryption has become an essential strategy for data security over the network. Encryption prevents sensitive data from getting hacked and misused by hackers for illegitimate purposes, and it also helps protect your computer from viruses and accompanying system vulnerabilities. To keep your personal data secure and away from potential malicious users, DiskStation Manager (DSM) adopts an encryption technology called Advanced Encryption Standard (AES), by storing your data in an encrypted format with a set of encryption keys. In addition, DSM provides share-level AES 256-bit encryption to block unauthorized access attempts.

This article will guide you through creating encrypted shared folders on your Synology NAS and remind you of important encryption issues.

What is AES?

Advanced Encryption Standard (AES) is a specification for the encryption of electronic data. AES has been adopted by the U.S. government since 2001 and is now widely used over the world. The algorithm implemented by AES is symmetric so that encrypting/decrypting requires the same encryption key. Without the key, the encrypted data is inaccessible to ensure information security.

1. Before you start

This article assumes that you have done the following tasks for your Synology NAS:

  • Hardware installation for Synology NAS
  • Software installation for Synology DiskStation Manager (DSM, a web-based operating system of Synology NAS)

Please refer to the Hardware Installation Guide for more information about hardware and software installation. You can also see Synology NAS User's Guide (available at Synology's Download Center) to learn more about topics related to this article.

2. Encrypt shared folders

As an administrator (DSM admin, or a DSM user belonging to the administrators group), you can either encrypt a shared folder when creating it or encrypt existing shared folders.

Note:

  • We strongly suggest exporting and saving the encryption keys for your encrypted shared folders.
  • Without the key, it is impossible to crack the encryption and access the data even if you remount the drives on other devices.

2.1 Create encrypted shared folders

  1. Go to Control Panel > Shared Folder > Create. After entering the Name of the shared folder, click Next and do the following:
    • Tick Encrypt this shared folder, and enter the encryption key in the Encryption key and the Confirm key fields.
    • Tick Add encryption key to Key Manager to manage keys of shared folders, decrypt multiple encrypted shared folders at the same time, and also be able to automatically mount the encrypted shared folders on startup.

    2. Note:

    • If you have not initialized your Key Manager, please initialize it first. Detailed steps for initializing and managing keys in Key Manager will be described in Manage Keys of Encrypted Shared Folders.
    • If Add encryption key to Key Manager is unticked, when your Synology NAS starts up next time, you will need to click Mount (at Control Panel > Shared Folder > Encryption), and enter or import the encryption key to mount the folder for access.
  2. After clicking Next, follow the remaining steps of the wizard to finish creating an encrypted shared folder.
  3. After you click Apply in Confirm Settings, a pop-up window will remind you if the encryption key is lost, there is no way to recover the encrypted data. Once you click Yes in the pop-up window, the encryption key will be saved automatically. We strongly suggest that you save the key.
  4. Save the encryption key by doing either of the following:
    • After you click Yes in the pop-up window, the encryption key will be saved automatically.
    • You can also go to Control Panel > Shared Folder > Encryption to export the key again.

Note:

  • The following system default shared folders cannot be encrypted: web, photo, music, video, surveillance, download, NetBackup.
  • Encrypted shared folders cannot be accessed via NFS, and will remove the previously set NFS rules if there is any.

2.2 Encrypt existing shared folders

  1. Go to Control Panel > Shared Folder. Choose a shared folder you want to encrypt and click Edit > Encryption.
  2. Follow the steps in Create encrypted shared folders to finish encrypting shared folders.

Note:

  • If you tick Enable file compression when creating a shared folder, encrypting this shared folder will disable file compression, and you cannot compress the folder anymore.

3. Configure access permissions for users and groups

To edit users' or groups' access permissions to the encrypted folder, go to Control Panel > Shared Folder, select the encrypted folder, and click Edit.

  1. Click Permissions and select any of the following from the drop-down menu:
    • System internal user: Anonymous FTP/Presto/WebDAV users. You can allow anonymous FTP/Presto/WebDAV users to access a shared folder here. Refer to Synology NAS User's Guide (available at Synology's Download Center) for more information about anonymous FTP/Presto/WebDAV.
    • Local users: Synology NAS users (including guest). Refer to Synology NAS User's Guide for more information.
    • Local groups: Synology NAS groups (Two groups are created by default: administrators and users). Refer to Synology NAS User's Guide for more information.
  2. Configure access permissions for users or groups:
    • Read only: Allow users or groups to view the content of the shared folders.
    • Read/Write: Allow users or groups to view and make changes to the content of the shared folders.
    • No access: Deny users or groups any access to the content of the shared folders.

    Note:

    • In case of permission conflicts, the permission priority is as follows: No access > Read/Write > Read only.
    • When you create a new shared folder, if the access permissions for admin (or a user belonging to the administrators group) are set to No access, then admin (or the user belonging to the administrators group) will only be able to see the folder by going to Control Panel > Shared Folder.
  3. Click Advanced Permissions for the following options if needed:
    • Disable directory browsing
    • Disable modification of existing files
    • Disable file downloading
  4. Click OK.

4. Manage keys of encrypted shared folders

You can use Key Manager to do the following things:

  • Manage keys of shared folders
  • Decrypt multiple encrypted shared folders at the same time
  • Enable encrypted shared folders to be mounted automatically on startup

Term explanation:

  • Key store: A key store can be system partition or any external devices supported by Synology NAS and is used as a physical key to decrypt shared folders.
  • Cypher: A cypher is a method of encrypting keys of shared folders, which store in system partition or external devices. Key Manager provides two types of cyphers:
    • Passphrase: Keys encrypted by a passphrase can be decrypted by anyone who knows the passphrase. (This option will be enabled only when the encryption keys are stored in external devices.)
    • Machine key: Keys encrypted by a machine key can only be decrypted by the bound Synology NAS.

4.1 Initialize Key Manager

If your Key Manager has not been initialized yet, please do the following:

  1. Go to Control Panel > Shared Folder > Action > Key Manager.
  2. Select an external device or system partition as the Key Store Location.
  3. Enter a passphrase in the Passphrase field if you store keys in an external device. Otherwise, you can only use Machine key for cypher.
  4. Tick Migrate encryption keys to this key store for all existing shared folders mounted automatically (recommended) to store those keys in a selected key store location and manage them.

Security Suggestions:

  • For safety and management purpose, it is safer to store the encrypted data and the corresponding key in different devices. We recommend selecting an external device as the Key Store Location to enhance your data security.

4.2 Add keys to be managed in Key Manager

You can add a new encryption key and manage it with Key manager by doing the following:

  1. Make sure that you have initialized Key Manager.
  2. Go to Control Panel > Shared Folder > Action > Key Manager.
  3. Click Add.
  4. Select an encrypted shared folder.
  5. Select the cypher for the encryption key. You can select either Passphrase or Machine key when the keys are stored in an external device. If the keys are stored in system partition, you can only select Machine key to encrypt keys.

    6. Note:

    • If you choose passphrase to encrypt keys of shared folders, the shared folders cannot be automatically mounted on startup of your Synology NAS.
  6. Enter or import the encryption key.
  7. Click OK to save the settings.

5. Decrypt the encrypted shared folders

There are two ways to decrypt an unmounted encrypted shared folder.

5.1 Automatically decrypt encrypted shared folders on startup

  1. Make sure that you have initialized Key Manager.
  2. Go to Control Panel > Shared Folder > Action > Key Manager.
  3. Click Add.
  4. Select an encrypted shared folder.
  5. Select Machine key to encrypt the key stored in the key store.
  6. Click OK to save the settings.
  7. Tick Mount on boot to automatically decrypt encrypted shared folders on startup.

5.2 Manually decrypt encrypted shared folders

If the encrypted shared folder you wish to access is unmounted, please do the following:

  1. Go to Control Panel > Shared Folder and click the encrypted shared folder.
  2. Click Encryption > Mount to mount the folder.
  3. Enter or import the encryption key, or enter the passphrase for Key Manager.
  4. Click OK.

If the keys are stored in Key Manager, you can also decrypt encrypted shared folders by doing the following:

  1. Go to Control Panel > Shared Folder > Action > Key Manager.
  2. Select one or more encrypted shared folders, click Mount to decrypt them.

6. Troubleshooting

  • Why can't I see my encrypted shared folders after my Synology NAS starts up?

    • Go to Control Panel > Shared Folder > Action > Key Manager, and check if Mount on Boot is ticked. If not, encrypted shared folders will not be mounted automatically on startup to block all access. You will need to manually mount the encrypted folders by entering or importing the encryption key for access. Please refer to Encrypt shared folders for more detailed instruction.

  • Why do file transfer speeds become slower when the shared folder is encrypted?

    • It is normal for encrypted shared folders to have slower file transfer speeds. Encryption can significantly increase the CPU workload and decrease the data transfer rate. To have enhanced throughput, you may consider a Synology NAS equipped with a hardware encryption engine:

    • To find a qualified Synology NAS, visit Synology's official website, select your product, and check the hardware specifications.
    • To compare multiple models and their performances, visit Synology's official website and go to Home > Products > Compare All NAS Models.
  • Can I back up the encrypted shared folders on my Synology NAS and restore the data within the folders?

    • Yes. Note that these restored encrypted shared folders will be unmounted and will not be automatically mounted on startup. You will have to enter or import the encryption key to mount these folders again. Considering this backup/restore issue, we strongly recommend exporting and saving the key every time you create an encrypted shared folder. Otherwise, it will be impossible to access or restore the encrypted data without the key. Please refer to Encrypt shared folders for more detailed instruction.
